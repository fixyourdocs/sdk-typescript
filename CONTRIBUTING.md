# Contributing

Thanks for your interest in the Docs Feedback Protocol.

## Developer Certificate of Origin (DCO)

Every commit must be signed off with a Developer Certificate of Origin
line. The DCO is a lightweight assertion that you wrote the contribution
or otherwise have the right to submit it under the project's licence.
Full text and rationale: <https://github.com/torvalds/linux/blob/master/Documentation/process/submitting-patches.rst#sign-your-work-the-developer-certificate-of-origin>.

Sign off your commits with `git commit -s`:

```sh
git commit -s -m "Your commit message"
```

This appends a trailer to the commit message:

```
Signed-off-by: Your Name <your.email@example.com>
```

## Contributor License Agreement (CLA)

In addition to the DCO, every contributor must sign the FixYourDocs
Individual Contributor License Agreement (based on the Apache ICLA v2.2)
the first time they open a pull request. The CLA-assistant bot will
comment on your PR with a one-click sign-off link. See the CLA text at
<https://github.com/fixyourdocs/.github/blob/main/CLA.md>.
